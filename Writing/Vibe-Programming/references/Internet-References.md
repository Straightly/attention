## Wikipedia

### 2. Vibe Coding
**URL:** https://en.wikipedia.org/wiki/Vibe_coding  
**Type:** Encyclopedia entry  
**Last updated:** 2025

**Key Information:**
- Term coined by Andrej Karpathy (February 2025)
- Definition: Coding approach relying on LLMs, allowing programmers to generate code via natural language
- Karpathy's description: "fully giv[ing] in to the vibes, embrac[ing] exponentials, and forget[ting] that the code even exists"
- Key distinction: User accepts AI-generated code without fully understanding it

**Reception and Use:**
- Kevin Roose (NY Times): Created "software for one" applications
- Y Combinator (March 2025): 25% of Winter 2025 startups had 95% AI-generated codebases
- Wall Street Journal (July 2025): Entering commercial use cases
- Fast Company (September 2025): Reports of "vibe coding hangover"

**Limitations:**
- Mischaracterization of software development
- Quality of code and security issues
- Task complexity challenges
- Debugging difficulties

**Relevance:** Comprehensive overview of term origin, adoption, and challenges.

### 8. Verification Loop Principle
**Source:** Andrej Karpathy

**Key Principles:**
1. Make verification EASY and FAST
2. Keep AI "on a tight leash" to increase probability of successful verification
3. Agents are overrated: "An agent generating 1000 lines of code is of no use to me if I have to read through them and verify it"

**Design Questions:**
- Can an LLM "see" all the things the human can?
- Can an LLM "act" in all the ways a human can?
- How can human supervise and stay in the loop?
- How does the diff look?

**Relevance:** Core design philosophy for effective AI-assisted development.

## Historical Context

### 15. Natural Language Programming (Historical)
**Examples:**
- COBOL (1959): Attempted English-like syntax
- SQL (1970s): Declarative, English-like queries
- AppleScript (1993): Natural language scripting

**Difference from Vibe Programming:**
- Historical: Constrained syntax that resembled English
- Vibe Programming: True natural language with LLM interpretation

**Relevance:** Shows vibe programming as culmination of decades-long trend.

## Critical Perspectives

### 16. Gary Marcus Response
**Source:** Cognitive scientist Gary Marcus (response to Kevin Roose)

**Critique:** Roose's enthusiasm stemmed from reproduction, not originality. AI presumably trained on existing code for similar tasks.

**Relevance:** Raises questions about creativity vs. pattern matching in AI-generated code.
