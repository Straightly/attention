# Vibe Programming: A Personal Journey

## Overview

This folder contains a comprehensive presentation paper on **vibe programming** based on 25 days of empirical experience (October 16 - November 8, 2025) using the Windsurf/Cascade AI coding assistant.

## What is Vibe Programming?

"Vibe programming" is a term coined by Andrej Karpathy in early 2025 to describe a coding approach where developers use natural language to describe intent and let LLMs generate code, "fully giving in to the vibes" and "forgetting that the code even exists."

## Key Statistics

- **Duration**: 25 days
- **Prompts documented**: 107
- **Time invested**: 20-25 hours
- **Output**: 1 web app, 4 major papers (1,000+ lines), infrastructure improvements
- **Estimated traditional development time**: 100+ hours
- **ROI**: 4-5x productivity improvement

## Key Findings

1. **Vibe programming works best with small, verifiable increments** rather than large autonomous agents
2. **Communication skills become more important than coding skills**
3. **AI systems exhibit human-like forgetting patterns**, revealing architectural limitations
4. **Cross-domain capability expands dramatically**—no need to be an expert in every technology
5. **Verification becomes the bottleneck** as AI capability increases
6. **The "broken pencil principle" applies to AI**: External memory is more reliable than internal memory
